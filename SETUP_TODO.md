# Manual setup — things only you can do

Everything code-side is done. These are the account/deployment steps that need your
credentials. Work top to bottom; the app works without them (the AI prompt bar just
shows "not configured" and offers the bring-your-own-key option until step 5 is done).

## 1. Create an Anthropic API key (~5 min)

1. Go to <https://console.anthropic.com> and sign in / create an account.
2. **Settings → Billing**: add a payment method and — important — set a **monthly
   spend limit** (e.g. $5). This is the hard backstop; even if every other guard
   fails, you cannot spend more than this.
3. **Settings → API Keys → Create Key**. Name it `arcadesoundfx-worker`. Copy it —
   it is shown once.

## 2. Set up Cloudflare (~10 min, free)

1. Create a free account at <https://dash.cloudflare.com/sign-up> (no domain or
   credit card needed — Workers free tier covers this many times over).
2. Install and authenticate the CLI:

   ```sh
   cd worker
   npm install
   npx wrangler login   # opens a browser
   ```

3. Create the KV namespace used for rate limiting:

   ```sh
   npx wrangler kv namespace create RATE_LIMITS
   ```

   This prints an `id`. Paste it into `worker/wrangler.jsonc` where it says
   `REPLACE_WITH_KV_NAMESPACE_ID`.

## 3. Store the API key as a Worker secret

```sh
cd worker
npx wrangler secret put ANTHROPIC_API_KEY
# paste the key from step 1 when prompted
```

The key never appears in the repo, the frontend bundle, or the Worker source —
it lives only in Cloudflare's secret store.

## 4. Deploy the Worker

```sh
cd worker
npx wrangler deploy
```

The output ends with your Worker URL, something like
`https://arcadesoundfx-ai.<your-subdomain>.workers.dev`.

Sanity-check it (should return JSON, not an error):

```sh
curl -s -X POST https://arcadesoundfx-ai.<your-subdomain>.workers.dev \
  -H 'content-type: application/json' \
  -H 'origin: https://tteuber.github.io' \
  -d '{"prompt": "a sad laser powering down"}'
```

## 5. Point the app at your Worker

Open `services/llmConfig.ts` and set:

```ts
export const WORKER_URL = 'https://arcadesoundfx-ai.<your-subdomain>.workers.dev';
```

Commit and push — the existing GitHub Pages workflow deploys the site as usual.
(The URL is public by design; the secrets stay in Cloudflare.)

## 6. Tune the limits (optional)

Defaults are conservative. In `worker/wrangler.jsonc` under `"vars"`:

| Variable | Default | Meaning |
| --- | --- | --- |
| `DAILY_LIMIT_PER_IP` | `10` | Requests per visitor per UTC day |
| `DAILY_LIMIT_GLOBAL` | `250` | Requests across all visitors per UTC day |
| `MODEL` | `claude-haiku-4-5` | Model the Worker uses |

Worst-case math at the defaults: 250 requests/day x ~$0.002/request ≈ **$0.55/day**
(~$16/month) if someone deliberately maxes the global cap every day; realistic
portfolio traffic is pennies per month. The Anthropic spend limit from step 1
caps it absolutely.

After editing, redeploy: `cd worker && npx wrangler deploy`.

Dev notes: `worker/worker-configuration.d.ts` is generated by `npm run types`
(inside `worker/`). If you ever regenerate it, first create `worker/.dev.vars`
containing `ANTHROPIC_API_KEY = "anything"` — that file is gitignored and only
tells the type generator (and `wrangler dev`) that the secret exists.
